Links

Usage notes

  1. How to make CSS-pure accessible drop-down menu?

    I use focus-within pseudo-class as if Scott O'Hara does.

  2. What're the cutting-edge techniques to underline links?

    I prefer using text-decoration-* properties after reading Benjamin Woodruff and John D. Jameson posts.

  3. What's the most annoying thing in links for people using the keyboard?

    outline: none; rule. I always highlight link focus status!

  4. Finally, should we put target="_blank" or not?

    I do it almost NEVER. Users must control your website behavior. I must mark such links. The vulnerability is underestimated.

    You open the viral page from Facebook. The viral page changes previous tab with the viral URL.

    I use <a rel="noopener noreferrer" target="_blank" …>.

  5. What else about CSS security?

    :visited selector was tricky. Now it's okay. So you're to decide.

Images

Usage notes

  1. I know <img> what else?

    HTML5 introduced <figure> element. This is standard for images.

    MDN Example:

    <figure>
    <img src="/media/examples/elephant-660-480.jpg"
        alt="Elephant at sunset">
    <figcaption>An elephant at sunset</figcaption>
</figure>

  2. I'm confused with alternative text. W3C doesn't help. What and where should I write?

    What — I've read WebAim article. They don't give abstract examples but teach you writing proper alt texts.

    Where — Scott O'Hara`s article shows us the way.

    Final example:

    <figure role="figure" aria-label="repeat figcaption content here">
    <!-- figure content. if an image, provide alt text -->
    <figcaption>
        Caption for the figure.
    </figcaption>
</figure>

<!--
aria-label for macOS VoiceOver + Chrome
role="figure" for IE11.

IE11 needs an accessible name (provided by aria-label).
If not for the fact VO + Chrome doesn't support an
accessible name from aria-labelledby, that attribute
would have been preferred / pointed to an ID on 
the <figcaption>.
-->

  3. How to make my images print-friendly?

    I use filter property. So images are black and white.

    Example:

    @media print {
    .figure__img {
        filter: gray;
        -webkit-filter: grayscale(1);
        filter: grayscale(1);
    }
}

Forms

Usage notes

What makes forms inaccessible?

  1. CAPTHCA. It's 100% inaccessible for disabled people, 100% inaccessible for ad blocker and VPN users and 100% annoying for anyone else.

    Imagine you buy yogurt and shop assistant asks you if you're robot

    Why do we forgive such distrust of sites?

    If you cannot live without making your users` life harder, please find CAPTCHA alternatives.

  2. Fields are not accessible via Tab. Label form controls properly and hardly ever change Tab order.

  3. Hidden Errors. Website's telling me I'm wrong and should redo work. Who obeys who? Provide polite notifications.

  4. Is it a checkbox or a bird? Default browser's fields appearance can not fit your website design. But don't forget they are the design standard. You can decorate it, but not to alter.

  5. Forms don't remember us.

    — How much is banana yogurt? — 43 — and cherry? — 48 — then banana one — What is banana?

    Why do we forgive sites for their forgetfulness?

    The form should “remember” everything user put in it. Do not force the user to do all the work again after an error, reloading or accidentally closing the page.

  6. Uncertainty. If I check it, do I enable or disable it?

    Action form controls labels — always a verb — and it must correspond to the action that will occur when a button is pressed, box is checked, etc.

Tables

Usage notes

  1. Are tables flexible?

    Yes, they are. You can find the torrent of table variants.

  2. How to use them properly nowadays?

    The only rule: make them look as if they are in a book and read the getting started post by Adrian Roselli.

  3. Can tables be responsive?

    Sure. And the answer can be flexible. There are data best to be presented as a list, image, separate table. The Responsive Accessible Table article by Adrian Roselli helps us.

  4. What is the list of steps for me? And how can I use JavaScript?

    A Complete Guide to the Table Element by Chris Coyier covers all the table questions.
